Pre Requisites

  • Vagrant
  • Vagrant AWS plugin
  • Amazon Web Services  (AWS) account

Fresh Deployment

The following steps will provision the LABEL application on AWS.

  1. Install the latest release of Vagrant.

  2. Install Vagrant AWS provider

    vagrant plugin install vagrant-aws

  3. Login to AWS management console and create the following resources 

    1. Elastic IP 1 for development server  (AWS_DEV_ELASTIC_IP)
    2. Elastic IP 2 for QA/UAT server  (AWS_QA_ELASTIC_IP)
    3. Elastic IP 3 for CI (jenkins) server  (AWS_CI_ELASTIC_IP)
    4. Create a new key pair (AWS_KEYNAME, AWS_KEYPATH)

  4. Set the following environment variables with appropriate values for AWS account and the resources created above


    Variable NameDescription
    AWS_ACCESS_KEY_IDAWS Key
    AWS_ACCESS_SECRETAWS Secret key
    AWS_KEYNAMEAWS Key name created in step 3.d
    AWS_KEYPATHPath of AWS key downloaded during step 3.d
    AWS_DEV_ELASTIC_IPAWS elastic IP created in step 3.a
    AWS_QA_ELASTIC_IPAWS elastic IP created in step 3.b
    AWS_CI_ELASTIC_IPAWS elastic IP created in step 3.c
    AWS_WIN_PASSWORDA password to be used for the windows machine to be created


  5. Check out the code Git

    git clone https://github.com/semanticbits/label-priv.git LABEL

  6. Run Vagrant

 

cd vm
vagrant up

 

 

Once the development environment is up, all deployments to DEV and QA/UAT are automated via Jenkins builds.

DEV Deployment

The DEV deployment process is automated, and runs on a nightly basis, or as part of the label-dev build in Jenkins. The deployment script is here.

QA/UAT Deployment

The label-uat build is used to deploy to the QA/UAT environment from a given tag.
