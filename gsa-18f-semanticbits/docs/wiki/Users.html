In order to really understand what people need, we have included "users" from the beginning of the project.  These are:

  • Paul Baumgartner, senior project manager in "real life"
  • Vasudev Chatti, project manager in "real life"
  • Ramakrishna Gundala, senior software engineer in "real life"

These users were involved from the very beginning of the project.  While the "idea" for the project originated from the "perceived need" of the Product Manager, we refined the requirements and functionality through interaction with the users in the following key engagement meetings:

  • S2 - Wireframe Review with Users
  • S4 - Mockup Review with Users
  • S8 - Usability Review with Users

A number of usability changes were requested in each of the above user sessions, which were incorporated into wireframes, mockups, and the final application.  The majority of the major changes that impacted usability were incorporated.  Due to the tight timeline, these usability changes were not documented as user stories, but rather incorporated into existing work.  The final deployment contains all the usability changes that were approved by the Product Manager.
